# GStack Parallels

Use this when adapting broader GStack logic to Abundance Network skills.

## Core Parallel

```text
GStack:
unclear idea -> guided diagnosis -> durable artifact -> specialized reviews -> ship/test -> feedback -> learning

Abundance:
unclear civic or policy idea -> guided diagnosis -> Abundance Systems Memo -> specialized reviews -> circulate/test -> feedback -> learning
```

## Front Door

| GStack | Abundance |
| --- | --- |
| `office-hours` | `diagnose` |
| design doc | Abundance Systems Memo |
| startup/product goal | civic action and implementation goal |

## Follow-On Reviews

| GStack | Abundance |
| --- | --- |
| `plan-ceo-review` | `strategic-foresight-review` |
| `plan-eng-review` | `implementation-mechanics-review` |
| `plan-design-review` | `public-translation-review` |
| `review` | `evidence-review` |
| `qa` | `feedback-loop-check` |
| `ship` | `publish-circulate-test` |
| learnings/context | `pattern-library-entry` |

## Adaptation Rule

Preserve GStack's logic unless the civic context changes the success condition, evidence standard, artifact, or feedback loop.
