# Abundance Systems Memo Skill Draft

Generated from office-hours discussion on 2026-06-04.

## Purpose

Adapt the GStack office-hours skill for the Abundance Network.

Instead of ending with a product plan for something to sell, the skill should produce an action plan that colleagues can promote, implement, test, and refine in line with abundance-oriented views.

The output should help a mixed coalition stitch together insight across domains. Housing people should learn from government IT failures. Permitting reformers should learn from procurement reform. Mobility advocates should recognize land-use and institutional bottlenecks. The skill should make the machinery visible across niches.

The skill should be backed by a growing knowledge library of historical, theoretical, and domain-specific material. That library should inform diagnosis without turning the memo into a book report.

Source architecture:

- Canonical sources provide durable lenses: books, essays, talks, reports, and theoretical frameworks.
- Messy live material provides the actual diagnosis: agency docs, meeting minutes, bills, budgets, procurement documents, environmental review records, litigation, reporting, case studies, interviews, field notes, and implementation data.

Canonical sources provide the map. Live diagnostic material checks the terrain. The memo should be explicit about which claims come from durable theory, which come from case-specific evidence, and which remain uncertain.

## Audience

The v1 audience is colleagues in the Abundance Network, including:

- Local YIMBY organizers
- City staffers
- State legislative aides
- Nonprofit policy leads
- Motivated citizens
- Government execution and IT reformers
- Housing, land-use, energy, transit, infrastructure, and mobility advocates

The audience is intentionally mixed because each person has a niche. The value is in stitching together cross-domain insight and recognizing common system mechanics.

## Inputs

The v1 skill should accept:

1. A concrete local, state, or federal problem the user wants to move.
2. A policy idea the user already believes may help.

Adjacent but separate future stacks:

- A cross-domain pattern library for wins and failures.
- A topic-exploration stack for understanding an abundance area more deeply.

## Output

The primary output is an Abundance Systems Memo with two layers:

1. Internal diagnosis: frank, causal, strategic, and willing to name weak assumptions.
2. Public-facing message: a derived advocacy frame suitable for allies, officials, media, or coalition use.

The internal layer must come first. The public layer should not be produced until the diagnosis has been run.

The public-facing layer should be a translation table, not a detached advocacy one-pager.

Each public claim must trace back to the internal diagnosis:

- Internal finding
- Public claim
- Evidence supporting the claim
- Audience most likely to respond
- Risk of overclaiming
- Safer wording if confidence is low

The goal is forceful communication without laundering uncertainty into slogans.

## Governing Rule

Do not advocate before understanding.

Advocating a policy because it sounds abundance-aligned, without understanding how it enters real institutions, is motivated reasoning. The skill should be abundance-aligned but not abundance-cheerleading.

## Diagnostic Center Of Gravity

The memo starts with in-depth diagnosis. It asks:

- What system is this policy entering?
- What incentives already govern that system?
- Where does authority actually sit?
- Who benefits from friction?
- Who absorbs the cost of delay?
- What second-order reactions should we expect?
- If the policy were an organism, how would it react in this environment?

## Full-Stack Diagnosis

The skill should force all three diagnostic lenses before choosing a dominant explanation:

1. Institutional mechanics: agencies, processes, procurement, litigation, veto points, staffing, budgets, authority.
2. Political economy: interest groups, concentrated benefits, diffuse costs, incumbents, fiscal incentives, patronage, risk allocation.
3. Cultural and ideological defaults: risk aversion, scarcity mindset, proceduralism, distrust of building, anti-state assumptions, anti-growth assumptions.

The point is not to force false balance. The point is to investigate before deciding which layer matters most.

The abundance book/framework lenses should behave the same way: all major lenses are lightly present in every run, then the memo goes deep only where the weighted causal model shows explanatory weight.

## Weighted Causal Model

Root causes are often multicausal. The memo should produce a weighted causal model instead of pretending there is one magic cause.

For each cause, track:

- Causal weight: how much this factor explains the problem.
- Tractability: how addressable it is from where the group actually sits.
- Time horizon: whether it can move in weeks, months, years, or decades.
- Confidence: what evidence supports the weight.
- Feedback signal: what would prove the model is wrong or drifting.

The memo may identify a high-weight cause that is hard to move and a lower-weight cause that is tractable. It should be honest about that distinction.

Example: fragmented authority may be the biggest cause, while procurement language may be the most tractable near-term lever. Fixing procurement should not be presented as solving the larger governance problem, but it can reduce one failure mode and teach the group where deeper blockage remains.

## Evidence Ladder

The skill should use an evidence ladder:

1. Start with desk research: public docs, news, statutes, agency pages, reports.
2. Identify field intelligence needed: implementers, advocates, staffers, affected people.
3. Identify document archaeology needed: budgets, procurement docs, environmental review, meeting minutes, litigation, org charts.
4. Mark uncertainty explicitly when action must proceed before full evidence is available.

Uncertainty is allowed. Fake certainty is not.

## Two-Pass Workflow

The skill should support a two-pass workflow.

Pass 1: hypothesis memo.

- No live research is required yet.
- The memo is explicitly labeled provisional.
- Causal weights are tentative ranges, not conclusions.
- The goal is to expose the likely system structure, name what must be investigated, and give the group a useful starting model.

Pass 2: evidence-backed revision.

- The skill gathers or ingests live diagnostic material.
- Causal weights can be revised, narrowed, or rejected.
- The action portfolio is updated based on what the evidence shows.
- Any remaining uncertainty is kept visible in the uncertainty register.

The first pass creates momentum. The second pass prevents the first story from becoming unearned certainty.

## Feedback Loops

Tight feedback is a core principle. The memo should define feedback signals that prevent drift and hollow progress.

Hollow progress means the paper policy exists, the press release is out, or the coalition feels satisfied, but the actual system behavior has not changed.

Every action plan should include:

- What should change in observable behavior.
- How soon we expect to see signal.
- Who will know first if implementation is drifting.
- What data, stories, documents, or operational facts should trigger revision.

## Action Portfolio

The action plan should track the whole system while forcing an actionable focus.

It should include:

1. Structural bet: the deeper reform implied by the diagnosis.
2. Near-term lever: the thing this group can actually push now.
3. Learning experiment: the fastest way to test whether the causal model is right.
4. Watchlist: signals that the intervention is drifting, being captured, watered down, litigated, or becoming performative.

The action focus should not pretend to solve the whole system if it only addresses one tractable piece.

## Instinct Audit

When a user brings a policy idea they already believe in, the skill should ask:

- What do you already believe is true here?
- Why do you believe it?
- What would make you change your mind?

If the instinct survives contact with the system map, incentives, institutions, and feedback loops, the memo can preserve the goal and redesign the implementation path.

If the mechanism is weak, the memo should split the conclusion:

- The goal may be right.
- The mechanism may be weak.

If the mechanism is fundamentally broken, the skill should challenge the policy itself and offer alternatives.

## Proposed Memo Structure

1. Problem frame
2. Initial instinct audit
3. Full-stack root cause diagnosis
4. System map
5. Incentive analysis
6. Policy-as-organism test
7. Weighted causal model
8. Evidence ladder and uncertainty register
9. Failure modes
10. Strategic foresight
11. Action portfolio
12. Feedback loops and drift signals
13. Internal strategy memo
14. Public translation table
